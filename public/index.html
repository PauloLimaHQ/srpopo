Autonomous Mode

Sr. Popo will drive every Ready task in this workspace end-to-end — dispatch each run (which plans, builds, tests, self-reviews, and opens a PR), then merge the PR and move the task to Done once it’s green — until the budget is spent. You can start it with an empty queue: it stands by and picks up each task the moment it lands in Ready. Runs are unattended: tool-permission prompts are turned off for them, so make sure each task is configured to run without a human.

Models

Add your own models on top of the built-in ones (Sonnet, Opus, Haiku, Fable). They show up in every model picker — New Task, Brief an Idea, and Linear import. A custom model is handed to claude --model as-is, and its environment variables are set only for runs on that model.

Use this for an Amazon Bedrock model: set the model id and add CLAUDE_CODE_USE_BEDROCK=1 plus AWS_REGION. AWS credentials come from your machine's environment (e.g. an AWS_PROFILE) — don't paste secrets here. Sr. Popo never uses an Anthropic API key (ANTHROPIC_API_KEY is always stripped).

Usage

Local spend tracking, computed from every session's own cost/token report — the same figures the claude CLI shows you, not a subscription bill (Sr. Popo never uses an API key). Codex sessions report tokens only, with no dollar figure, so they show a cost of —. Nothing leaves your machine.

Remote Access (LAN)

Open and control this board from another device on the same local network — a laptop, tablet, or phone on the same Wi-Fi. The board looks and works exactly like it does here.

Local network access
Anyone on your network who has the link and token can control your tasks and repositories. Keep this off unless you need it, and only on networks you trust. Traffic is unencrypted (HTTP) — LAN use only, not the public internet.
Pair a device

Open this URL from another device on the same Wi-Fi/network. The token is included, so it signs in automatically.

Rotating the token signs out every currently-paired device.
